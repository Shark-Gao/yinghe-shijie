# AGENTS.md - 硬核视界 AI 工作规范

## 项目目标

维护「硬核视界」面向多平台分发的硬核知识短视频项目。B 站不再是唯一发布目标；围绕同一优质海外源视频，按平台特点制作并分发不同版本。

## AI 助手职责

- 推荐符合定位的 YouTube 视频
- 维护视频库和制作记录
- 优化标题、封面文案
- 遵守项目规则

## 项目技能位置

- 所有为「硬核视界」新增或维护的项目专属技能，必须存放在本项目的 `skills/` 目录下。
- 不将项目专属技能安装到用户级 Codex 技能目录；通用系统技能不受此限制。

## 当前制作流程 V1.1

推荐视频 → Telegram 下载 MP4 → 剪映生成中英字幕 → 按平台制作版本 → 多平台发布 → 记录数据并复盘

## 短视频生成偏好

- 用户提到“生成短视频”时，默认使用项目内的 `yinghe-short-video` 技能：以原始视频和字幕为输入，产出平台适配的中文短视频。
- 不使用 `local-text-to-video` 技能，也不采用“静态图片轻微动效”的方式替代短视频制作。

## 多平台分发原则

- 采用“一源多剪、多平台分发”：同一源视频可根据不同平台的节奏、开头、标题和封面制作不同版本。
- 抖音、快手、小红书优先制作 30–90 秒、开头强钩子、信息密度高的版本；标题突出冲突、规模、成本、竞争或结果。
- B 站作为较完整内容、系列化内容和账号品牌沉淀的阵地，不再用 B 站的推荐逻辑决定全部选题和剪辑方式。
- 小红书强化“冷知识 + 可收藏”的表达；抖音、快手强化前几秒吸引力和视觉奇观。
- 每次发布后记录各平台的播放、完播、互动等数据，据此调整选题、时长、包装与优先发布平台。

## 选片原则

- 优先少真人或无真人出镜
- 优先 3D 动画、工程可视化、纪录片、地图动画
- 不重复推荐同一个源视频
- 保持主题系列化
- 军工装备为高优先级题材；在候选质量相当时，优先航母、潜艇、战机、导弹防御、无人装备等少真人、强工程可视化内容。

## 国内化选题思路

- 不只看外网视频是否硬核，还要判断能否包装成国内观众熟悉的热门选题。
- 优先选择能对应国内同类 UP 主常见方向的素材：大国工程、产业竞争、AI 应用、军工装备、超级机械、地理冷知识、灾难复盘。
- 降低纯冷门原理题材的优先级，除非它能转化成明确的大众问题，例如“芯片为什么难造”“AI 为什么需要 GPU”“航母内部如何运转”。
- 中文标题要优先突出冲突、结果、规模、成本、稀缺性或现实用途，而不是直译英文标题。

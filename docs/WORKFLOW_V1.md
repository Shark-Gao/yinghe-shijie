# 硬核视界 V1.1 工作流程

当前固定流程：

```text
1. ChatGPT 按国内化选题思路推荐 YouTube 视频
2. 复制链接到 Telegram 下载原始 MP4
3. 导入剪映
4. 自动生成中英字幕
5. 按发布平台制作对应版本
6. 多平台发布，并记录各平台数据

## 多平台制作与发布

采用“一源多剪、多平台分发”。选片以国内受众与素材表现力为依据，不再由 B 站的推荐偏好决定全部选题和剪辑方式。

| 平台 | 内容角色 | 制作重点 |
|---|---|---|
| 抖音、快手 | 获取曝光与验证选题 | 优先 30–90 秒；前几秒给出冲突、结果或视觉奇观；节奏紧凑。 |
| 小红书 | 获取收藏、搜索与长尾流量 | 强化“冷知识 + 现实问题 + 可收藏”的标题和封面表达。 |
| B 站 | 系列化内容与品牌沉淀 | 可使用相对完整的版本；强化系列感、背景信息和观看价值。 |

同一源视频可以制作不同开头、时长、标题、封面和文案。中文包装始终优先突出冲突、结果、规模、成本、稀缺性或现实用途。

发布后记录播放、完播、互动、收藏/转发等数据，定期比较同一题材在各平台的表现，用数据调整后续选题与剪辑版本。
```

## 推荐前判断

推荐外网视频前，先判断它能不能转成国内同类 UP 主常见的热门表达：

- 是否属于大国工程、产业竞争、AI 应用、军工装备、超级机械、地理冷知识、灾难复盘。
- 是否能提炼出明确标题钩子：冲突、规模、成本、稀缺性、现实用途、为什么很难。
- 是否有足够强的画面承载力：3D 动画、地图、工业现场、工程剖面、数据可视化。
- 是否能避免纯冷门原理感，让普通科技观众在标题阶段就知道为什么值得看。

## 当前原则

- 暂时不加入中文旁白
- 保留原视频英文音频
- 优先保证制作效率和稳定产出
- 后续根据数据决定是否升级流程
